# Vattenkontot MVP v0.5

Dygnsvis uppdaterad prototyp för hydrologiskt informerad konsumtionskommunikation för Vombsjön/Vombverket.

## v0.5 innehåller

- Livehämtning av Vombverkets leverans från Sydvatten.
- Livehämtning av vattennivå i Vombsjön från SMHI station Vombsjön övre.
- Säsongsbaserad klassning av tillgång utifrån historiska percentiler.
- Första HydroNu/S-HYPE-proxy för påfyllnadsutsikt: Björkaån/Eggelstad, SUBID 103.
- Supabase-lagring av Sydvatten-avläsningar för att senare kunna använda rullande 24-timmarsmedel.

## Supabase-tabell

Skapa tabellen i Supabase SQL Editor:

```sql
create table if not exists public.sydvatten_readings (
  id bigint generated by default as identity primary key,
  created_at timestamptz not null default now(),
  source text not null default 'sydvatten',
  vomb_lps numeric,
  ring_lps numeric,
  raw jsonb
);

create index if not exists sydvatten_readings_created_at_idx
on public.sydvatten_readings (created_at desc);
```

## Environment variables i Render

Lägg in följande under Render → Environment:

```text
SUPABASE_URL=<din Supabase Project URL>
SUPABASE_SERVICE_KEY=<din Supabase secret/service key>
```

Alternativt fungerar även namnen `SUPABASE_SECRET_KEY` eller `SUPABASE_SERVICE_ROLE_KEY`.

Valfria variabler:

```text
SYDVATTEN_SAVE_INTERVAL_MINUTES=15
SYDVATTEN_FULL_DAY_MIN_HOURS=20
SYDVATTEN_FULL_DAY_MIN_READINGS=6
HOUSEHOLD_COUNT=200000
```

## Förbrukningstakt

I v0.5 sparas Sydvatten-avläsningar i Supabase. Innan tillräcklig historik finns använder appen fortfarande senaste avlästa Vombverket-värde. När minst cirka ett dygns historik finns använder API:t rullande 24-timmarsmedel som förbrukningstakt.

Appen visar därför både:

- Senast avläst: X l/s
- 24h-medel: samlas in / används
